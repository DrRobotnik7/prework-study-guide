HTML


This website is a great HTML resource MDN HTML

For the purposes of this section, brackets are to be replaced with angled brackets in VS Code

The words “element” and “tag” are commonly used interchangeably in the web development world, but it is important to know the difference. An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.

Try to use semantic elements as much as possible. They do what they say i.e. header and img, which makes your HTML easier to read by other developers and the browser.


  • The code starts with the (!DOCTYPE html) declaration, which tells the browser what document type it should expect when it is opened.
  • The (html) opening tag represents the root, or start, of our HTML document. Inside the (html) tag is an attribute of lang="en", which tells the browser the language for the webpage
  • The (head) element contains metadata, data about our webpage that the browser uses to effectively serve our webpage to users such as such as the character set, description, keywords, author, and viewport
  • The (header) element can contain a navigation menu and/or introductory content
  • The (body) element represents the visible content shown to the user.
  • The (h1), (h2), (h3), (h4), (h5), and (h6) elements represent the level of heading a given text block represents. Each page should only contain maximum one (h1) heading
  • The (main) element contains the main part of the website vs the body element which contains everything visible on the page
  • The (p) element represents a paragraph or block of text.
  • The (img) element is self-closing e.g. (img src="https://static.fullstack-bootcamp.com/prework/module-03/lesson-01/cute-pandas-02.jpg" alt="Cute Pandas" width="520" height="346")
  • The (ul), (ol), and (li) elements represent unordered lists, ordered lists, and list items.
  • The break element (br /) introduces a page break and looks like (br /)
  • The anchor element (a) creates hyperlinks to a webpage e.g. (a href="https://nationalzoo.si.edu/animals/giant-panda")More About Giant Pandas(/a)

CSS


Adding external CSS style sheet to asset file


  1. cd "insert repo name here" then cd assets: moving to the assets folder, obviously repo must have that folder first
  2. touch style.css: adds file to asset file
  3. git status: make sure you are in the main branch, if not then navigate there with git checkout main
  4. git checkout -b feature/add-css: adds a new branch to work on
  5. In VSCode navigate to new style.css file in assets folder
  6. Copy and paste whatever rules you want in your style sheet
  7. The * selector applies to ALL elements on the page
  8. Where possible use multiple selectors in one rule instead of having same rule multiple times - see DRY rule (do not repeat yourself). This reduces the number of lines of code.
  9. Add the following in the head section, after the meta elements (link rel="stylesheet" href="./assets/style.css")

Useful information


  • This website is a great CSS resource MDN CSS
  • Assigning colors in CSS can be done in many ways, including by semantic references such as gray or blue. We can also refer to a color by its hexadecimal code, or hex #
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Use class selectors to apply rules to all members of a class e.g. (p class="footer") in HTML then a .footer selector in style.css sheet

Git


Common commands


  • git clone "paste copied SSH value here": clone repository to local environment
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: sync so local main branch mirrors latest version of GitHub main branch - otherwise you might be working on outdated code
  • git add -A: adds modifications made to the staging area, so these will be pushed up on next commit
  • git commit -m "insert description of what you are adding here"
  • git push origin "enter name of branch you are working on here": moves changes to the cloud, it will now appear in GitHub
  • git checkout "enter name of branch here": moves to the branch you want to work on

GitHub Workflow


First, make repo on GitHub website. Now, set up local environment:

  1. git clone "paste copied SSH value"
  2. git status
  3. git checkout -b branch-name: make a new branch
  4. git status: check you are working in the new branch
  5. git pull origin main

Now make changes to your new branch:

  1. git add -A
  2. git commit -m "insert description of what you are adding here"
  3. git pull origin main: again, sync the local main with the cloud main branch
  4. git push origin "enter name of branch you are working on here": moves changes to the cloud, it will now appear in GitHub

Now do a pull request at GitHub to merge your changes from the new branch into the main branch in the cloud:

  1. Once merged delete the branch you were working on as it has now merged into the main branch
  2. Finally we need to sync our local environment back with the cloud so need to pull down the main branch. Go to the command prompt:
  3. git checkout main - moves to the main branch
  4. git pull origin main - syncs local copy with the updated branch on GitHub

JavaScript


Adding external JS sheet to asset file


  • Create script.js in assets file, same process as adding CSS file
  • In index.html, at the bottom under (footer) but above (body) add code (script src="./assets/script.js")(/script)
  • It goes at the bottom as it affects the HTML code, the browser reads code from top to bottom. So for JS to affect the HTML code, HTML needs to load first

Useful information


  • MDN JS Basics
  • MDN JS Data Structures
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false. All other values, including a Boolean with the value of true, are considered truthy.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • String i.e. letters need to be in quotation marks. Numbers and Boolean true/false values don't need to be.
  • In JS and many other programming languages, arrays are zero-indexed and sequential. The first item has an index of 0, not 1. The indices, or location identifiers, will always be whole numbers that increase by 1 each time.
  • As per the above, the below code will result in triangle because it has an index of 0. Square is 1, pentagon is 2 etc.
    var shapes = ["triangle", "square", "pentagon", "circle"];
    console.log(shapes[0]);
